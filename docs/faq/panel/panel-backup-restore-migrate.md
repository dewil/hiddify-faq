# Бэкап и перенос панели Hiddify

Бэкап в Hiddify - это один JSON-файл с настройками панели и пользователями. Он нужен, чтобы быстро поднять панель на новом сервере, если текущий заблокирован или сломался.

## Как сделать бэкап

В админ-панели откройте "Settings" -> "Backup" и нажмите кнопку получения файла бэкапа. Браузер сохранит JSON с настройками - выберите, куда его положить.

Бэкап также создается автоматически в трех случаях:

- Каждые 6 часов панель делает бэкап сама. Если включен [Telegram-бот](https://hiddify.com/manager/configuration-and-advanced-settings/How-to-set-up-and-use-Telegram-bot-on-Hiddify-panel/), файл отправляется в бот.
- При обновлении или переустановке панели бэкап создается перед операцией.

Если бот недоступен и веб-интерфейс не открывается, прошлые бэкапы лежат прямо на сервере:

```
/opt/hiddify-manager/hiddify-panel/backup
```

Скопировать файл на компьютер можно через scp:

```
scp myserver:/opt/hiddify-manager/hiddify-panel/backup/<файл>.json ./
```

где `myserver` - ваш SSH-алиас (или `root@203.0.113.10`).

## Восстановление

Установите панель на новом сервере. На странице "Quick-setup" ничего не настраивайте. Затем откройте "Settings" -> "Backup" и выберите ранее сохраненный JSON-файл. Включите нужные опции восстановления: чтобы поднять панель целиком (настройки и пользователей), отметьте все три и дождитесь завершения.

## Перенос на другой сервер

1. Сделайте бэкап на старом сервере (см. выше).
2. Установите чистую панель на новом сервере, "Quick-setup" не трогайте.
3. Восстановите из бэкапа все три опции.
4. Переключите DNS вашего домена на IP нового сервера. Например, в Cloudflare откройте записи `vpn.example.com` и впишите новый IP (`203.0.113.10`).

После обновления DNS панель и админ-ссылка поднимутся на новом сервере, клиенты продолжат работать по тому же домену.
