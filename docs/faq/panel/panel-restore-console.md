# Веб-восстановление панели Hiddify висит - как довести из консоли

Нажали "Восстановить" из бэкапа, прогресс крутится и не завершается, панель в полуживом состоянии. Хорошая новость: к этому моменту данные бэкапа обычно уже записаны в базу - зависает следующий этап, пересборка конфигов. Значит, восстановление можно довести до конца по SSH, не трогая веб больше.

## Порядок действий

1. **Зайдите на сервер по SSH** под root.

2. **Не нажимайте "Восстановить" в вебе повторно.** Если данные уже в базе, повторный запуск ничего не ускорит, а при [баге с extra_params](restore-hangs-extra-params.md) - заново сломает то, что вы почините.

3. **Проверьте, на чем падает сборка конфигов:**

    ```bash
    hiddify-panel-cli all-configs >/dev/null && echo OK
    ```

    - Напечатало `OK` - база в порядке, переходите к шагу 5.
    - Упало с `json.decoder.JSONDecodeError` - это битый `extra_params` после восстановления. Почините по статье [Восстановление виснет или падает с JSONDecodeError](restore-hangs-extra-params.md) и возвращайтесь к шагу 4.

4. **Пересоберите конфиги** (тот самый шаг, который висел в вебе; несколько минут):

    ```bash
    bash /opt/hiddify-manager/apply_configs.sh
    ```

5. **Проверьте результат:**

    - `hiddify-panel-cli all-configs >/dev/null && echo OK` печатает `OK`;
    - админ-ссылка открывается в браузере;
    - любой пользовательский конфиг подключается, подписка обновляется.

Все три пункта сошлись - восстановление завершено.

## Если восстанавливать нечего: где лежат бэкапы

Панель складывает автоматические бэкапы прямо на сервере:

```
/opt/hiddify-manager/hiddify-panel/backup
```

Если сервер живой, а нужен именно откат настроек - берите оттуда свежий JSON. Если поднимаете панель на новом сервере с нуля - это уже перенос, порядок в статье [Бэкап и перенос панели Hiddify](panel-backup-restore-migrate.md).

Держите копию бэкапа не только на самом сервере: файл, который лежит только на упавшей машине, при отказе диска не спасет. Проще всего включить встроенный Telegram-бот панели - тогда бэкап каждые 6 часов уходит вам в бот.

## Источники

Реальный кейс восстановления боевой ноды (июль 2026): веб-восстановление зависло на этапе пересборки, из консоли нода была поднята за один прогон `apply_configs.sh` после починки `extra_params`.
