# Happ на Android - прокси для выбранных приложений

В Android-версии клиента **Happ** есть функция **"Прокси для выбранных приложений"** (Per-app Proxy) - она задает, какие приложения ходят через VPN, а какие напрямую. Функция только для Android: на iOS/iPadOS система не дает выбирать трафик по отдельным приложениям, а на десктопе per-app нет вовсе (альтернативы - в [Раздельное туннелирование по приложениям на ПК](per-app-split-tunneling-desktop.md)).

Открывается в **Настройки -> Прокси для выбранных приложений**. Сверху три вкладки-режима, снизу - список установленных приложений с галочками.

## Главное: список приложений общий для всех трех вкладок

Частая путаница - кажется, что ВЫКЛ / ВКЛ / Обход это три разных списка. На самом деле **список с галочками один и тот же**, а вкладка меняет только то, как этот список трактуется. Отметьте приложения и попереключайте вкладки - галочки никуда не денутся, поменяется лишь их смысл.

## Три режима

- **ВЫКЛ** - функция выключена, VPN применяется ко всем приложениям без исключений. Галочки в этом режиме ни на что не влияют, но сохраняются: можно заранее собрать список приложений, а потом переключить режим.
- **ВКЛ** - через VPN идут **только отмеченные** приложения, все остальные подключаются напрямую. Режим "белого списка": VPN включается точечно для нужных программ.
- **Обход** - **отмеченные** приложения идут напрямую, в обход VPN, а все остальные - через VPN. Режим "черного списка": VPN на всем, кроме исключений.

## Что выбрать

- **Весь трафик под VPN, кроме российского** - режим **Обход**, отметить российские приложения (банки, Госуслуги, каршеринг, АЗС, авиакомпании, 2ГИС). Они пойдут напрямую с вашим реальным российским IP - иначе через зарубежный сервер такие приложения либо не работают, либо требуют лишних подтверждений. Это самый частый и удобный сценарий.
- **VPN только для пары приложений** (например, браузер и мессенджер) - режим **ВКЛ**, отметить только их.
- **VPN на все без исключений** - режим **ВЫКЛ**.

!!! note "Обход работает на устройстве, до сервера"
    Отбор приложений происходит на самом телефоне, еще до туннеля. То, что попало в "Обход", уходит напрямую с устройства и до сервера не доходит - значит серверные правила маршрутизации (в том числе geoip-разделение российских сетей) на него не распространяются. Поэтому в "Обход" добавляйте только то, что действительно должно работать с домашним IP, а тонкую маршрутизацию по сайтам оставляйте серверу и режиму региона (см. [Раздельное туннелирование: российские сайты напрямую](split-tunnel-ru-direct.md)).
