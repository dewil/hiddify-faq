# Раздельное туннелирование по приложениям на ПК - почему нет и чем заменить

Частый запрос: пустить через VPN только выбранные приложения (Discord, игру, локальный сервер), а остальное оставить напрямую. В десктопном клиенте Hiddify такой настройки нет - это не баг, а отсутствующая функция.

## Чего нет

Hiddify на Windows и macOS умеет маршрутизировать трафик только двумя способами:

- **Весь трафик через туннель** (режим VPN-сервис).
- **По региону** - российские сайты напрямую, остальное через VPN. Это настраивается параметром "Регион" и описано отдельно: [Раздельное туннелирование: российские сайты напрямую](split-tunnel-ru-direct.md).

Выбора "вот эти приложения - в туннель, остальные - мимо" (per-app) на десктопе нет. Такая функция есть только в Android-версии (там можно отметить приложения галочками). Запросы добавить ее на ПК в комьюнити повторяются давно, но в клиент она не завезена.

## Чем заменить на ПК

### Proxy-режим + проксификатор

Переключите клиент из режима VPN-сервис в режим системного прокси - тогда Hiddify поднимает локальный прокси, но сам трафик в него не заворачивает (как это влияет на систему - см. [Режим VPN или Proxy](vpn-vs-proxy-mode-windows.md)). Дальше внешний проксификатор (например Proxifier или ProxyCap) направляет в этот локальный прокси только нужные приложения по правилам. Это и дает выбор по приложениям, которого нет в самом Hiddify.

### Только браузер или отдельные сайты

Если задача уже - пустить через VPN конкретные сайты, а не приложения целиком, проще настроить прокси на уровне браузера (расширение-переключатель прокси с правилами по доменам), а систему оставить напрямую.

### Другой клиент с правилами маршрутизации

Подписку Hiddify (ту же ссылку) можно открыть в клиенте, который поддерживает гибкие правила маршрутизации по приложениям и доменам - например nekobox/NekoRay, Karing, Throne, v2rayN. Сам сервер и подписка не меняются, меняется только приложение на устройстве.
