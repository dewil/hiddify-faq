# Как настроить VLESS+Reality в Hiddify

VLESS+Reality - основной прямой протокол Hiddify Manager. Включен по умолчанию, но регулярно ломается из-за выбора SNI, "грязного" IP сервера и блокировок РКН.

## Симптомы

- Клиент подключился, в трее зеленый статус, но сайты не открываются, в логах: `reality verification failed`, `context deadline exceeded`.
- Работает только VLESS, остальные протоколы висят в таймауте.
- Reality-сканер в панели показывает "0 возможных вариантов" SNI.
- На Windows подключение проходит, интерфейс tun0 без сети, IP остается домашним.

## Почему это происходит

Reality маскирует трафик под TLS-хендшейк к чужому домену. Если SNI в списках РКН или сам сервер из заблокированной подсети, обманка не работает: РКН с осени 2024 целенаправленно бьет именно Reality. Часть сервисов требует чистого IP в ASN Cloudflare WARP, иначе Google и игры детектят корпоративный хостинг.

"Подключено, но нет интернета" на клиенте обычно значит: TUN не поднялся (нужен sudo на macOS, Service Mode на Windows), либо для tcp-direct в подписке прописан забаненный IP сервера.

## Что делать

- В админке отключите все протоколы кроме VLESS, Trojan, Reality - остальное под блокировкой нестабильно.
- В `Settings - Domains` для Reality-домена проверьте Direct Mode и SNI. Работают `s3.aeza.net`, microsoft.com; gmail/google под риском.
- Грязный IP (Hetzner, OVH) - спрячьте за promezhutochnyj relay с haproxy и в подписке принудительно укажите IP relay для Reality TCP домена.
- Включите WARP в режиме "Только заблокированные" - подменяет outbound-org на Cloudflare.
- На клиенте: Service Mode/Системный прокси, не TUN-экспериментальный. При ошибках auto/URLTest вручную выберите профиль.

## Если не помогло

- Добавьте `xhttp` или `grpc` транспорт - переживут точечные блокировки vless TCP.
- 12.3.0 ломала Apply Config и добавление доменов - обновитесь до 12.3.1.
